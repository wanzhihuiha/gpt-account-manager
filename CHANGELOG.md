# 更新记录

## 2026-06-03 dashboard-stats

- 新增工作区风险仪表盘，汇总封禁邮件趋势、今日/近 7 天/周期封禁数量、涉及邮箱、域名排行和最近封禁邮件。
- 新增 `/client-api/dashboard-stats` 只读接口，基于当前工作区缓存邮件、邮箱资产和刷新结果生成统计，不触发取信或删除。
- 顶部模块导航新增“仪表盘”入口，并补充中英文切换文案。

## 2026-06-03 generic-mail-provider

- 新增普通邮箱取信入口，支持 IMAP/POP3 自动推断以及 CloudMail/LuckMail/Inbucket 类 API 取信。
- 邮箱管理页支持普通邮箱导入、同步、筛选、导出和删除。
- 凭证刷新页的邮箱预检和登录取码链路支持普通邮箱，并补充对应中文错误分类。

## 2026-06-01 server-mail-cache

- 邮件正文和 HTML 改为写入服务端工作区缓存 `data/workspaces/<workspace>/messages.json`，浏览器不再保存完整邮件数组。
- 首页邮件列表改成从 `/client-api/messages` 分页读取，筛选、分页、单封删除、批量删除都直接操作服务端缓存。
- 收信接口只返回轻量结果：邮件数量、验证码列表和首个验证码，避免大批邮件触发浏览器 `localStorage` 配额错误。
- CPA 仓管页“收取验证码”不再写 `ctgptm.mail.messages`，只使用轻量验证码摘要。
- 刷新页邮箱验证改为使用 `codes / first_code / has_verification_code / message_count`，不再依赖完整邮件正文返回。

## 2026-06-01 fetch-background

- 邮箱读取改为后台任务：前端只启动任务并轮询状态，后端继续直接连接 Outlook / 临时邮箱 API 收信，避免网页长请求被 Nginx/Cloudflare 504 截断。
- 撤销整批 45 秒截止线；大量邮箱会按并发池继续跑完，不会把还没排到的邮箱统一标成取信超时。
- 前端遇到 504/HTML 网关错误时显示可读提示，不再把整页错误 HTML 当作业务错误展开。

## 2026-06-01 outlook-scroll

- 修复首页左侧邮箱清单固定高度下无法滚动的问题，选择 20 / 50 条时不再裁掉后续邮箱。
- Outlook 自动收信优先走 IMAP，并保留 Graph / Outlook API 作为回退；Graph 401 不再把 IMAP 可用账号误标为失败。
- IMAP token 刷新顺序改为优先 `login.live.com`，兼容更多 Outlook 四段凭证。
- 前端收信方式统一规范为 `auto / graph / imap`，选择单一路径失败时会提示切回微软自动。

## 2026-06-01 final

- 增加轻量国际化脚本，默认中文，右上角可切换中文 / EN。
- 语言选择保存在当前浏览器，本地刷新和页面跳转后保持一致。
- 覆盖顶部导航、主要工作台按钮、筛选项、表头、常见提示和输入框占位文案。
- 国际化层只翻译界面固定文案，不处理用户邮箱、token、代理 URL 或日志里的业务数据。

## 2026-06-01

- 刷新页新增“验证邮箱”按钮，先独立检查取码邮箱是否可收件、当前是否能读到验证码邮件。
- 验证邮箱会自动使用当前工作区保存的真实取码凭证，避免把前端打码后的 Outlook/临时邮箱凭证误判为失败。
- 已验证失败或未验证的邮箱不会直接加入刷新队列，避免把坏邮箱带进 OAuth 刷新流程。
- 邮箱验证结果写回左侧邮箱状态：可收件、未发现验证码、Graph/IMAP 收信失败、临时邮箱 JWT 无效等。
- 邮箱管理页权限类型改为 `JWT`、`Graph/IMAP`、`Graph+IMAP+密码`，导入类型下拉只保留邮箱名称，不再误写成 OAuth。
- README 增加脱敏界面截图，关键邮箱、token、代理信息已打码。
- 邮箱管理页继续打磨：左侧邮箱管家改为浅色高对比，不再被全局玻璃主题冲淡。
- 邮箱管理页移除异常主分类；异常邮箱只在邮箱名旁标红“（错误）”。
- 新增独立“邮箱管理”页面：左侧分类统计，右侧表格管理，支持搜索、来源/分组筛选、分页、批量复制、批量分组、导出备份和批量删除。
- 邮箱管理页支持 Outlook 邮箱和临时邮箱导入；临时邮箱导入新增工作空间级接口，导入后刷新页也能同步到。
- 顶部功能导航增加“邮箱管理”入口，避免把邮箱资产管理继续堆在凭证刷新页里。
- 刷新队列顶部统计改成可点击筛选：队列、等待、执行中、成功、失败都能直接切换列表。
- 失败账号清理改为按“已选账号 / 当前失败筛选 / 全部失败”批量处理，不再只限制封禁、账号不存在等确定失败类型。
- 左侧已导入邮箱增加“移除所选”，可在成功/失败/需要接码筛选后批量移除对应取码资料和队列记录。
- 邮箱验证码、手机验证码都改为按需弹窗输入，默认继续自动收码，不再在页面常驻输入框。
- 正在执行或排队的账号增加“终止”，后端轮询邮箱码/手机码时会响应终止并停止等待。
- 长效手机池进一步压缩为折叠内紧凑布局，手动手机验证码改为弹窗填写。
- 凭证刷新改为队列优先：单个账号按钮只加入队列，统一由刷新页顺序执行，避免手动点击时绕过排队。
- CPA 异常扫描台和仓管页的刷新动作改为“加入刷新队列”，不再在原页面直接启动登录任务。
- 协议登录默认走邮箱验证码路径，避免把 Outlook/临时邮箱的取码凭证误当成 OpenAI 登录密码提交。
- 优化邮箱验证码链路：支持 passwordless 发码接口，按真实收信时间过滤旧验证码，并修正验证码提取误识别英文单词的问题。
- 增加 OAuth 手机二次验证状态机：识别 `phone_otp_select_channel` / `/phone-otp/select-channel`，进入短信通道后等待长效手机 API 或手动手机验证码。
- 长效手机接码池支持批量导入、1 对 1 绑定、按手机号尾号匹配账号，并把手动输入的手机验证码回传给正在运行的登录任务。
- 代理处理增加粘性会话：支持 `{session}` 占位符，并对部分代理服务自动追加 session；同一账号执行前会复检出口 IP 是否稳定。
- 网络错误分类细化为 DNS 失败、代理超时、代理 TLS 中断、代理连接失败、代理出口不稳定等大类，减少“登录失败”这类笼统提示。
- 刷新失败自动重试扩展到可恢复的代理、DNS、风控和 OAuth 步骤异常；确定失败类账号继续保持不可重试。
- sub2api / CPA 导出继续保持成功结果里的 `auth_file` 结构，成功刷新记录保存到工作区 `refresh_results.json`。
